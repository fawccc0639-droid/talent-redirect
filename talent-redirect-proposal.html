一、问题诊断

在传统 B端 ATS（招聘管理系统）中，HR 对终面淘汰候选人执行"不通过"操作后，候选人数据即进入沉默池，系统不做任何后续干预。核心痛点有三：

  1. 人才错杀：候选人可能因单项技能未达阈值而被淘汰，但其未被评估的跨界能力（如前端候选人具备 95 分 WebGL + 85 分金融估值）完全流失；
  2. 沉没成本不可控：每流失一位高质量候选人，企业平均损失约 ¥2,500 获客成本（招聘渠道费 + HR 工时 + 面试官时间）；
  3. 雇主品牌折损：候选人收到冷冰冰的拒信后对雇主好感度下降，影响长期人才池建设。

二、方案设计

2.1 核心架构

系统采用 B端 SaaS 三栏布局（Sidebar 导航 + 智能流转中心 + C端预览），以 HR 为主用户、候选人复活页为预览输出：

模块功能
智能流转中心左栏候选人列表 → 中栏 AI 错位诊断报告 → 右栏 C端复活页实时预览
策略与模板配置路由规则开关、邀请模板管理、触达渠道配置
ROI 大屏本月复活人数、挽回成本、cNPS 提升等运营数据

2.2 多维加权匹配算法

不同于传统关键词匹配，系统采用能力向量加权评分模型：

  • Must-Have 加权：逐项计算 候选人得分/岗位阈值 × 权重，归约到 0~1；
  • 跨界能力加成：若岗位开启 crossDomainBonus，候选人非本职高价值技能额外贡献 bonus；
  • 元能力匹配：学习敏锐度、认知灵活性等通用素质形成 +0~5% 调节项。

实测：张同学投递常规前端岗仅得 60%（React 75 < 阈值 85），但路由至「数字资产可视化专家」得分 94%（WebGL 95 + FinTech 83 完美命中）。

三、AI 工具选型理由

工具用途选型理由
CodeBuddy (Deepseek-V4)全栈开发、代码生成支持复杂逻辑重构、多文件并行编辑，上下文窗口长，适合 SaaS 后台开发
Tailwind CSS (CDN)UI 布局 & 样式原子化 CSS，快速构建专业级 B端界面，无需构建工具
ECharts 5.5 (CDN)雷达图、柱状图国产成熟可视化库，深色主题适配好，支持动态 dispose/rebind
Vercel (GitHub 集成)公网部署每次 Git Push 自动部署，零配置，HTTPS 自动签发

四、关键配置

  • 候选人多维画像：12 项能力评分（7 硬技能 + 5 跨界特征） + 4 项元能力指标，结构化 JSON；
  • 岗位需求向量：mustHave（skill · minScore · weight）+ niceToHave（bonusWeight）+ crossDomainBonus 开关；
  • 打字机效果：递归 setTimeout 逐字输出，`typewriter-cursor` CSS 伪元素实现闪烁光标；
  • ECharts 窄容器适配：雷达图精简至 4 项核心指标 + 别名映射，`echarts.init(dom, null, { width, height })` 强制尺寸；
  • 滚动隔离：右栏 `flex-1 overflow-y-auto` + 父级 `min-h-0`，扫描线 overlay 纯 CSS 实现。

六、效果评估

  • HR 操作效率：从"手动翻看其他岗位 JD → 邮件邀约"约 20 分钟/人，降至系统一键流转 < 30 秒；
  • 人才利用率：AI 识别 Top 15% 高价值淘汰者并自动匹配跨部门岗位，避免人才错杀；
  • 候选人体验：从冰冷拒信变为个性化打字机推荐 + 雷达图复盘，Mock 数据显示 cNPS 提升 +42%；
  • 经济回报：按每月复活 12 人、人均获客成本 ¥2,500 计算，月度挽回 ¥30,000 沉没成本。
